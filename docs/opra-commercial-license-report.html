1. 요약 결론

질문 검토 결과 AuralTune 적용 판단
OPRA 데이터를 상업 앱에 사용할 수 있는가? 가능. OPRA README와 CC BY-SA 4.0 모두 상업적 사용을 허용한다. 유료 앱 또는 인앱 결제 앱에도 사용할 수 있다.
앱 코드를 공개해야 하는가? 일반적으로 불필요. OPRA 데이터와 앱 코드를 분리하면 앱 코드는 독자 라이선스를 유지할 수 있다. OPRA-derived 데이터만 CC BY-SA로 표시하고 앱 코드는 별도 소유로 유지한다.
수정 없이 쓰면 문제가 없는가? 사용 자체는 가능하지만 attribution, 라이선스 링크, 변경 여부 표시, 추가 제한 금지 조건은 유지된다. 정보 화면, EQ 상세 화면, EULA 예외 조항을 함께 넣어야 안전하다.
OPRA URL을 직접 호출해도 되는가? 상업 제품은 공식 문서상 자체 mirror/cache가 권장된다. 서버 캐시 또는 앱 배포용 snapshot 파이프라인을 두는 것을 권장한다.
최종 판단: OPRA 데이터는 상업적 앱 배포에 사용할 수 있다. 다만 AuralTune은 OPRA 데이터를 자체 독점 데이터처럼 보이게 하면 안 되며, 사용자가 OPRA 데이터의 CC BY-SA 권리를 행사할 수 있다는 점을 약관과 UI에서 명확히 분리해야 한다.

2. OPRA 프로젝트 개요

정식명 OPRA, Open Profiles for Revealing Audio
성격 헤드폰 제품 정보와 EQ compensation curve를 정리해 소프트웨어가 쉽게 사용할 수 있도록 배포하는 오픈 데이터베이스
운영 주체 Roon Labs가 저장소와 포맷, 문서, tooling, ingestion cleanup에 기여하며 커뮤니티 PR 방식으로 확장
목표 사용자 개인 사용자, 오픈소스 프로젝트, 상업 제품 모두
현재 배포 방식 GitHub 저장소의 dist/ 산출물과 Roon Labs의 Cloudflare 캐시 미러

OPRA는 AutoEq처럼 EQ를 생성하는 알고리즘 프로젝트라기보다, 제품 정보와 EQ 프리셋을 정리해 앱이 소비하기 쉬운 데이터베이스로 배포하는 프로젝트다. 즉 AuralTune 관점에서는 음향 처리 엔진이 아니라 데이터 소스다.

4. 데이터 수집 및 캐시 전략

방식 공식 권장 상업 앱 적합성 권장 판단
Roon Labs 미러 사용 오픈소스, 비상업, 개인 사용에 권장 상업 제품에서는 장기 운영 의존성 이슈가 있다. 개발, 테스트, 내부 QA용으로 사용
자체 mirror/cache 상업 제품에 권장 가장 적합하다. 트래픽, availability, 버전 통제를 자체 관리할 수 있다. 운영 배포 권장
GitHub raw 직접 호출 최신 결과 확인, 개발, 디버깅에 한정 권장 rate limit과 availability 리스크가 있다. 운영 앱에서는 피한다.

권장 수집 플로우

OPRA upstream
  -> scheduled mirror job
  -> immutable snapshot storage
  -> validation and license metadata injection
  -> AuralTune API or app-bundled snapshot
  -> local Room cache
  -> EQ detail attribution UI
상업 제품에서는 OPRA의 dist/ 폴더를 자체 인프라에 mirror/cache하는 방식을 채택하는 것이 가장 안전하다. 앱은 자체 서버 또는 버전 고정 snapshot을 사용하고, 원본 링크와 라이선스 링크를 함께 제공한다.

5. AutoEq와 OPRA의 관계

항목 AutoEq OPRA
핵심 역할 측정값 기반 EQ 결과 생성 및 results 배포 제품 정보와 여러 작성자의 EQ preset을 앱 친화적으로 정리
데이터 성격 results/INDEX.md, ParametricEQ 텍스트, WAV, PNG 등 JSONL 기반 vendor/product/eq 통합 카탈로그
관계 OPRA 초기 데이터의 주요 원천 AutoEq 데이터를 ingest/cleanup해 출발했으며 추가 작성자 협업을 지향
AuralTune 적용 기존 AutoEq importer 유지 별도 OpraRepository로 추가하는 것이 적합

6. 라이선스 구조

대상 라이선스 상업적 의미
OPRA 코드와 도구 MIT 도구 코드 사용, 수정, 배포 자유도가 높다.
제조사, 제품, EQ 데이터 CC BY-SA 4.0 상업 사용 가능. 단 attribution, Share-Alike, 추가 제한 금지 조건을 지켜야 한다.
상표, 로고, 제3자 권리 별도 권리 가능 CC 라이선스가 특허, 상표, 초상권, 개인정보 권리를 자동으로 허용하지는 않는다.
이 문서는 제품 기획과 구현을 위한 기술 검토이며 법률 자문이 아니다. 실제 유료 출시 전에는 약관, 데이터 번들링, 앱스토어 DRM, 국가별 데이터베이스권 이슈를 법무 검토에 넘기는 것을 권장한다.

7. 상업적 사용 상세 검토

CC BY-SA 4.0은 자료를 복사, 재배포, 변형할 수 있게 허용하며 상업 목적 사용도 허용한다. 따라서 OPRA 데이터를 수정 없이 앱에 표시하거나 EQ 적용에 사용하는 것은 원칙적으로 가능하다. 문제는 사용 가능 여부가 아니라 배포 방식과 고지 방식이다.

상업 사용 시나리오 가능 여부 조건 권장 구현
OPRA 데이터를 수정 없이 유료 앱에서 사용 가능 OPRA와 preset author attribution, CC BY-SA 링크, no endorsement 문구 필요 앱 정보 화면과 EQ 상세 화면에 고정 표기
OPRA JSONL을 앱에 번들 가능 데이터 파일이 CC BY-SA임을 사용자가 알 수 있어야 한다. 번들 파일 버전, 원본 URL, 라이선스 URL 저장
OPRA 데이터를 자체 서버에서 mirror/cache 가능 재배포에 해당하므로 OPRA 데이터의 라이선스와 원본 출처를 유지한다. 상업 제품용 기본 방식으로 채택
OPRA EQ 값을 내부 포맷으로 변환 주의 순수 기술 변환은 CC 법문상 adapted material이 아닐 수 있으나, 데이터베이스권과 파생 DB 논점이 생긴다. 변환 데이터도 OPRA-derived로 표시하고 원본 값 보존
OPRA EQ 값을 조정해 AuralTune 최적화 preset으로 배포 가능하나 조건 강화 수정 사실 표시와 동일 또는 호환 라이선스 적용 필요성이 커진다. OPRA 원본 preset과 AuralTune modified preset을 분리
OPRA 데이터와 독점 데이터를 하나의 DB로 병합해 판매 고위험 OPRA 포함 portion과 파생 데이터베이스에 Share-Alike 논점이 생긴다. OPRA-derived 테이블을 독립 저장하고 내보내기 가능하게 설계

상업 배포에서 지켜야 할 네 가지

조건 제품에서 필요한 조치 권장 표기 위치
출처 표시 OPRA, preset author, 원본 링크, 라이선스 링크를 제공 EQ 상세, 앱 정보, 오픈소스 라이선스 화면
라이선스 고지 OPRA 데이터가 CC BY-SA 4.0임을 명시 정보 화면, EULA, 데이터 소스 화면
추가 제한 금지 앱 약관이 OPRA 데이터 재사용 권리를 제한하지 않는다는 예외 조항 삽입 EULA, 개인정보/약관 페이지
보증/공식 인증 오해 방지 AuralTune이 OPRA, Roon Labs, preset creator의 공식 제품이 아니라고 표기 정보 화면, 앱스토어 설명, EQ 상세

8. Share-Alike 적용 판단

Share-Alike는 OPRA 데이터를 기반으로 수정, 변환, 병합한 파생 자료를 공유할 때 같은 또는 호환 라이선스를 유지해야 한다는 조건이다. 앱 코드 전체가 자동으로 CC BY-SA가 되는 것은 아니지만, OPRA 데이터와 그 파생 데이터는 별도 관리해야 한다.

행위 Adapted Material 가능성 Share-Alike 대응
원본 JSONL을 그대로 제공 낮음 attribution과 라이선스 고지 중심
앱 표시/재생을 위한 필드 파싱 낮음에서 중간 기술 변환으로 기록하고 원본 ID와 값을 보존
EQ 값을 반올림, 재정렬, 내부 band 모델로 저장 중간 변환 사실 표시, OPRA-derived 라이선스 유지
EQ 곡선을 재계산하거나 gain/Q를 조정 높음 수정 사실 표시, 동일/호환 라이선스 적용
OPRA와 자체 데이터를 합쳐 새 통합 DB로 배포 높음 OPRA portion 분리, 파생 DB 라이선스 검토
실무적으로는 OPRA 데이터를 opra_vendor, opra_product, opra_eq처럼 별도 namespace로 저장하고, 자체 데이터와 병합된 view만 UI에 제공하는 구조가 안전하다.

9. 리스크와 회피 방안

리스크 영향 권장 회피 방안
attribution 누락 라이선스 위반 가능성 필수 필드를 DB schema에 넣고 UI 테스트에서 누락 검증
앱스토어 DRM과 추가 제한 사용자가 CC 자료 권리를 행사하지 못한다고 해석될 수 있음 EULA에 OPRA 데이터 라이선스 예외 조항을 명시
Roon/OPRA 공식 인증 오해 상표, 표시광고, 파트너십 오해 위험 endorsed, certified, sponsored가 아님을 반복 표기
데이터 품질 또는 EQ 적용 사고 사용자 불만, 청취 품질 문제 원클릭 reset, preamp 확인, clipping guard, disclaimer 제공
로고/제품 이미지 권리 CC 데이터와 별도의 상표권 또는 제3자 권리 가능성 초기 버전은 텍스트와 EQ 숫자 중심으로 사용, 이미지 사용은 별도 검토
