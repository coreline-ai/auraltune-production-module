# implement_20260701_130412.md

작성 일시: `2026-07-01 13:04:12 +09:00`

이 문서는 **인앱 플레이어에 Media3 `MediaSessionService`를 도입**하여 잠금화면·알림 트랜스포트, Bluetooth·이어폰 미디어 버튼, 백그라운드 재생을 제공하는 개발 범위를 고정한다. EQ 정확도 경로(`AuralTuneAudioProcessor` 인라인)는 무변으로 유지한다.

## 개발 목적
- 잠금화면·알림에서 재생/일시정지/이전/다음/시크, **BT·이어폰 버튼**, 오디오 포커스·becoming-noisy 표준 처리.
- 앱이 백그라운드로 가도 **재생 지속** + 알림 상시 표시(제목·아티스트·앨범아트 자동).
- 현재 ViewModel 스코프의 ExoPlayer를 **서비스/앱 스코프로 이관**하되, 기존 재생·큐·EQ·스펙트럼 동작을 그대로 보존.

## 개발 범위
- `androidx.media3:media3-session` 추가.
- `AuralTuneMediaService`(신규, `MediaSessionService`) — ExoPlayer(기존 `RenderersFactory`+`AuralTuneAudioProcessor` 구성 그대로) + `MediaSession` 소유, Media3 기본 알림 제공.
- `engine`·`SpectrumAnalyzer`를 `ServiceLocator` 앱 싱글턴으로 승격(서비스·ViewModel 공유).
- `MusicPlayerController`를 ExoPlayer 직접 소유 → **`MediaController` 연결** 래퍼로 전환(비동기 연결 + 콜드스타트 버퍼링).
- 매니페스트 서비스 선언 + 포그라운드/알림 권한, `POST_NOTIFICATIONS` 런타임 요청.

## 제외 범위
- Android Auto/Wear 전용 UI, 미디어 라이브러리 브라우징(`MediaLibraryService`), 캐스트/원격 재생 — 이번 범위 아님(`MediaSessionService`만).
- 재생목록 편집/큐 UI 변경, 새 트랜스포트 기능(크로스페이드·슬립타이머 등) — 별도 트랙.
- EQ/프로파일/스펙트럼 알고리즘 변경 없음(위치만 앱 스코프로 이동, 동작 동일).
- 앨범아트 추출 로직 변경 없음(기존 파이프라인 재사용; 알림은 세션 메타데이터에서 자동).
