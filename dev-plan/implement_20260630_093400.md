# implement_20260630_093400.md

작성 일시: `2026-06-30 09:34:00 +09:00`

이 문서는 인앱 플레이어(T1)에 **반복/셔플 재생 옵션**을 추가하고, 콜드스타트 첫 재생 시맨틱을 **"현재 곡만 재시작"**으로 확정하는 개발 범위를 고정한다. 새 DSP/디코더 기능이 아니라, ExoPlayer(Media3)가 이미 제공하는 `repeatMode` / `shuffleModeEnabled`를 노출·영속·UI 연결하는 작업이다.

## 개발 목적

- 사용자가 재생 모드를 선택할 수 있게 한다: **랜덤(셔플) / 1곡 반복 / 전체 반복 / 한번만(리스트 1회)**.
- 전송부(transport)의 이전/다음 버튼 **좌우**에 셔플·반복 버튼을 적절한 아이콘으로 배치한다.
- 선택한 모드를 DataStore에 영속하고 앱 재시작 시 복원한다.
- 콜드스타트(복원된 곡이 끝 위치)에서 재생 시 **현재 곡만 처음부터** 재시작한다(큐 0번으로 점프하지 않음).

## 개발 범위

- `SettingsStore`에 `repeatMode`(Int: OFF/ONE/ALL) + `shuffleEnabled`(Bool) 영속 추가.
- `MusicPlayerController`에 모드 set/cycle/toggle 메서드, `PlaybackUiState` 확장, 리스너, 복원 적용 추가.
- 플레이어 화면 transport에 셔플(prev 왼쪽)·반복(next 오른쪽) 버튼 추가, 상태별 아이콘/하이라이트.
- 콜드스타트 재시작 경로 `seekTo(0, 0L)` → `seekTo(0L)`(현재 곡 0초).

## 제외 범위

- 새 DSP/디코더/하이레이트 재생 경로는 추가하지 않는다(별도 백로그).
- 미니플레이어에는 모드 버튼을 추가하지 않는다(공간 제약 — 풀 플레이어에만).
- 커스텀 셔플 순서/큐 편집 UI는 추가하지 않는다(ExoPlayer 기본 셔플 사용).

## 모드 매핑

| UI 옵션 | 동작 | ExoPlayer |
|---|---|---|
| 한번만 (리스트 1회) | 리스트 끝까지 1회 재생 후 정지 | `REPEAT_MODE_OFF` |
| 전체 반복 | 리스트 끝나면 처음부터 반복 | `REPEAT_MODE_ALL` |
| 1곡 반복 | 현재 곡 반복 | `REPEAT_MODE_ONE` |
| 랜덤 | 셔플 재생(반복과 독립 토글) | `shuffleModeEnabled = true` |

- 반복 버튼: `OFF → ALL → ONE → OFF` 순환. 셔플 버튼: 독립 토글.
- 아이콘: 반복 OFF/ALL = `Repeat`, ONE = `RepeatOne`, 셔플 = `Shuffle`. 활성 시 primary 틴트, 비활성 시 onSurfaceVariant.
