# implement_20260630_111731.md

작성 일시: `2026-06-30 11:17:31 +09:00`

이 문서는 **톤(Bass/Mid/Treble) EQ에 프리셋(내장 + 사용자 저장)** 기능을 추가하는 개발 범위를 고정한다. 그래픽/파라메트릭 EQ가 이미 갖춘 "내장 카탈로그 + 사용자 저장" 2단 구조를 톤에도 동일하게 적용한다. 톤 프리셋은 3게인([베이스,미드,트레블])만으로 완전히 기술되어 가장 단순하다.

## 개발 목적
- 톤 EQ에서 내장 시작점 프리셋을 빠르게 고르고, 현재 3게인을 사용자 프리셋으로 저장/불러오기/삭제.
- 그래픽 프리셋과 동일 UX(저장 버튼 + 불러오기 드롭다운 + 선택 표시 + dirty 해제), debug·release 공통.

## 개발 범위
- 신규 모델 `ToneEqPreset`(id/name/gainsDb[3]/타임스탬프).
- 신규 `ToneEqPresetCatalog`(내장 5종, id 접두사 `builtin:tone:`, 삭제 불가).
- `SettingsStore`: `userToneEqPresets` 저장/불러오기/삭제 + `selectedToneEqPresetId`.
- `AutoEqViewModel`: `toneEqPresets = 내장 + 사용자`, `saveToneEqPreset`/`loadToneEqPreset`/`deleteToneEqPreset`, 선택표시 + 슬라이더 변경 시 dirty 해제.
- `ToneEqCard`: 상단 프리셋 행(저장/불러오기) — 그래픽 프리셋 행 패턴 재사용.

## 제외 범위
- 엔진/주파수/Q 변경 없음(톤 게인 3개만 저장).
- 내장 프리셋 편집/삭제 불가(사용자 저장본만 삭제).
- 그래픽/파라메트릭 프리셋과 공유하지 않음(모드별 독립).

## 내장 프리셋 v0
| id | 이름 | [베이스, 미드, 트레블] |
|---|---|---|
| bass | 저음 강조 | [+5, 0, 0] |
| vocal | 보컬 강조 | [0, +3, -1] |
| bright | 청량 | [0, 0, +5] |
| vshape | V자 | [+5, -2, +5] |
| warm | 따뜻하게 | [+4, 0, -3] |
