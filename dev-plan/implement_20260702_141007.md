# implement_20260702_141007.md

작성 일시: `2026-07-02 14:10:07 KST`

이 문서는 플레이어 버튼으로 `원음 / EQ 적용 / 내 설정(또는 프리앰프 적용) / AuralTune / Android EQ` 상태를 바꿀 때 들리는 찌직음, 팝, 클릭 노이즈를 줄이기 위한 개발 범위를 고정한다. 목적은 새 음장 기능 추가가 아니라, 이미 구현된 EQ 전환 경로를 재생 중에도 부드럽게 만드는 것이다.

## 개발 목적

플레이어에서 EQ 관련 버튼을 눌렀을 때 오디오 처리 경로가 즉시 바뀌며 발생할 수 있는 클릭 노이즈를 제거한다.

핵심 원인은 전환 유형별로 다르다.

| 전환 | 현재 위험 지점 | 개선 방향 |
|---|---|---|
| AuralTune `원음 ↔ EQ 적용` | AutoEQ는 0.5초 fade가 있어 상대적으로 안전 | 기존 동작 유지, 회귀 확인 |
| AuralTune `EQ 적용 ↔ 내 설정` | Manual EQ chain이 즉시 on/off | native manual wet/dry ramp 추가 |
| AuralTune `프리앰프 on/off` | preamp gain glide는 이미 있음 | 기존 30ms glide 회귀 확인 |
| Android EQ `원음 ↔ EQ 적용 ↔ 프리앰프 적용` | `DynamicsProcessing` PostEQ/inputGain이 즉시 변경되거나 effect가 release됨 | neutral 유지 + 단계적 gain/EQ ramp |
| `AuralTune ↔ Android EQ` | native engine bypass와 OS effect attach/release가 재생 중 즉시 전환 | 전환 순서 제어, 필요 시 neutralize 후 release |

## 개발 범위

| 구분 | 포함 내용 |
|---|---|
| Native DSP | Manual EQ on/off에 wet/dry mix ramp 추가 |
| Android EQ | `DynamicsProcessing` backend를 Android 모드 안에서는 가능한 한 유지하고 neutral 상태로 전환 |
| Android EQ gain | input gain/headroom 변경을 짧은 단계 ramp로 적용 |
| Android EQ band | PostEQ band gain 변경을 짧은 단계 ramp로 적용 |
| UI 입력 | 전환 중 중복 클릭 방지를 위한 최소 transition guard 검토 |
| 검증 | 단위 테스트, native 테스트, PD20 실기기 전환 반복 확인 |

## 제외 범위

- 새 EQ 프리셋, 새 필터 타입, 맥시마이저, 리미터, MBC 추가는 하지 않는다.
- ExoPlayer 교체, Oboe/AAudio 재설계, 24/32-bit 출력 정책 변경은 하지 않는다.
- 외부 앱 audio session, system-wide EQ, session 0 처리는 하지 않는다.
- 음악 재생 큐, OPRA/AutoEQ 데이터 fetch, 검색 UI는 변경하지 않는다.
- 클릭음의 완전 자동 계측을 새로 만들지 않는다. 1차 검증은 로그/테스트/PD20 청감 확인으로 한다.

## 결정 기록

- 1차 패치 우선순위는 Android EQ 경로다. 최근 버튼/Android EQ 변경 이후 증상이 드러났고, effect attach/release/inputGain 즉시 변경이 클릭 노이즈 가능성이 가장 크기 때문이다.
- native AutoEQ on/off와 preamp glide는 이미 구현되어 있으므로, 우선 회귀 확인 대상으로 둔다.
- 새 음질 기능 추가가 아니라 전환 안정화가 목적이므로, 버튼 UI 대개편은 하지 않는다.
