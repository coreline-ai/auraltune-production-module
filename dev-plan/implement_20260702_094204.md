# implement_20260702_094204.md

작성 일시: `2026-07-02 09:42:04 +09:00`

이 문서는 Player / AutoEQ / OPRA 3개 탭 상세 코드 분석에서 나온 우선순위 항목을 다시 검토하고, 실제 동작 신뢰성과 상업 배포 감사성에 필요한 패치만 개발 범위로 고정한다. 새 기능, 운영 CI, 문서 미화, 선택 QA 확장은 이 문서의 범위가 아니다.

## 개발 목적

- AutoEQ/OPRA 프로파일이 native 엔진에 실제로 적용되었는지 UI와 상태 저장이 거짓 성공을 표시하지 않게 한다.
- `원음` 모드에서는 보정/프리앰프가 실제로 비가청 상태임을 UI도 동일하게 표시한다.
- AutoEQ의 `현재 사용` 동작이 현재 기기 선택 상태와 라우트 복원 상태를 일관되게 유지하게 한다.
- OPRA 번들 스냅샷의 manifest 검증과 provenance 표시를 상업 배포 기준에 맞게 강화한다.

## 개발 범위

- `DeviceAutoEqManager`의 `engine.updateAutoEq()` 반환값 처리.
- AutoEQ/OPRA 적용 실패 시 provider 전환, active 상태, 사용자 메시지의 실패 처리.
- Player / AutoEQ / OPRA 탭의 `현재 사용중`, source badge, preamp 적용 표시를 실제 `ListenMode`와 맞춤.
- AutoEQ `현재 사용` 재적용 시 현재 기기 선택 매핑 보존 또는 갱신.
- OPRA bundled source에서 manifest `sha256`, `generated_at`, `schema_version` 처리 강화.
- 위 항목을 검증하는 로컬 단위 테스트 또는 최소 빌드/수동 확인 기준 추가.

## 제외 범위

- seek 슬라이더 호출 최적화.
- `AuralTuneAudioProcessor.onFormatChanged` 중복 콜백 정리.
- AutoEQ `Clear cache`의 Room DB invalidate/refresh 확장.
- OPRA parser의 미래 schema type policy 확장.
- 문서 전반 정합화, README 배지 갱신, CI 운영 계획.
- query-driven online fallback, hi-res direct playback, T2/T3/외부 앱 적용.

| OPRA parser type policy | 후순위 | 현재 OPRA snapshot schema 기준 즉시 실패 증거 없음 | 아니오 |

- [x] 실패 로그는 debug/release 정책을 지키며 민감 정보 없이 남긴다.
