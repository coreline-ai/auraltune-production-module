# implement_20260630_105308.md

작성 일시: `2026-06-30 10:53:08 +09:00`

이 문서는 인앱 EQ에 **톤(Bass/Mid/Treble) 편집 모드**를 추가하는 개발 범위를 고정한다. 기존 `그래픽 / 파라메트릭` EQ 편집 모드 토글에 **`톤` 세그먼트를 1개 추가**하고, 선택 시 3밴드(베이스/미드/트레블) 슬라이더 카드를 같은 자리에 표시한다. 새 바텀탭이 아니라 EqMode 3번째 옵션이며, 그래픽/파라메트릭과 **상호배타**(단일 Manual 체인 공유)다.

## 개발 목적

- 20밴드 그래픽·자유 파라메트릭이 부담스러운 사용자를 위한 **가장 단순한 3-노브 톤 조절** 제공.
- 베이스(저역 셸프) / 미드(중역 피킹) / 트레블(고역 셸프) 게인만 조절.
- 기존 Manual 체인 적용 경로(`updateManualEq`)·응답 그래프·청취 모드·영속을 그대로 재사용.

## 개발 범위

- `EqMode`에 `TONE` 추가(3번째 모드, fromKey 관용 파싱 유지).
- `SettingsStore`에 `toneGains`(Float×3, [베이스,미드,트레블]) 영속/복원.
- `AutoEqViewModel`: `_toneGains` 상태, `manualResponseSpecs` combine에 톤 소스 추가, `currentManualSpecs`에 TONE 분기, `setToneGain`/`resetTone`, 복원/영속 collector.
- UI: `EqModeToggle` 3-세그먼트(`톤/그래픽/파라메트릭`), 신규 `ToneEqCard`(가로 슬라이더 3개 + 합성 응답 그래프 + 리셋).

## 제외 범위

- 엔진/네이티브 변경 없음(기존 LOW_SHELF/PEAKING/HIGH_SHELF 타입만 사용).
- 톤을 그래픽/파라메트릭 "위에 얹는" 마스터 스테이지는 하지 않음(상호배타 모드 유지).
- 주파수/Q를 사용자 조절하게 하지 않음(고정값 — 그게 톤 컨트롤의 단순성).
- 세로 페이더 UI는 이번 범위 외(가로 슬라이더로 구현; 후속 폴리시 가능).

## 모드 매핑 (엔진 무변경)

| 컨트롤 | 필터(nativeId) | 주파수 | Q | 게인 |
|---|---|---|---|---|
| 베이스 | Low Shelf(1) | 100 Hz | 0.70 | 슬라이더 ±한계 |
| 미드 | Peaking(0) | 1 kHz | 0.90 | 슬라이더 ±한계 |
| 트레블 | High Shelf(2) | 10 kHz | 0.70 | 슬라이더 ±한계 |

- 게인 |db| < 0.05 밴드는 spec에서 제외(그래픽/파라메트릭과 동일, 무음 0dB 필터 방지).
- 게인 한계는 기존 게인 한계 칩(±6/±12/±15/±20) 공유.

## GUI 정책

- EQ 편집 방식 토글: `[ 톤 ] [ 그래픽 ] [ 파라메트릭 ]` (톤이 첫 칩, 가장 쉬운 옵션).
- `ToneEqCard`: 가로 슬라이더 3줄(베이스/미드/트레블), 각 줄 `라벨 · 슬라이더 · +X.XdB`, 0dB 센터. 상단 합성 응답 그래프(프로파일+톤), `리셋` 버튼.
- "내 설정" 청취 모드에서 들리도록 변경 시 `ensureUserListenMode()`.
- 접근성 contentDescription(슬라이더/리셋).
