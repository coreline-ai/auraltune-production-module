# implement_20260702_113943.md

작성 일시: `2026-07-02 11:39:43 +09:00`

이 문서는 플레이어 내부 audio session에 `DynamicsProcessing` 기반 Android EQ 모드를 추가하는 개발 범위를 고정하고, 기존 AuralTune 엔진 경로와 동시에 적용되지 않도록 구현 순서를 관리한다.

## 개발 목적

- 플레이어 탭에서 사용자가 `AuralTune 엔진`과 `Android EQ` 처리 방식을 단순하게 전환할 수 있게 한다.
- `Android EQ` 모드는 Android framework `DynamicsProcessing`을 현재 ExoPlayer audio session에 attach해서 동작한다.
- `Android EQ` 모드에서는 AuralTune native AutoEQ/manual/preamp 처리를 bypass해서 이중 적용을 방지한다.
- 1차 목표는 메뉴를 복잡하게 만들지 않고, 플레이어 화면에서만 접근 가능한 저부하/호환 모드를 제공하는 것이다.

## 개발 범위

- `SettingsStore`에 처리 방식 저장값 추가.
- 플레이어/서비스가 공유할 처리 방식 상태 추가.
- `AuralTuneAudioProcessor`에 native engine bypass 모드 추가.
- 내부 플레이어 audio session에 `DynamicsProcessing`을 attach/release/apply하는 컨트롤러 추가.
- 현재 AutoEQ/OPRA 선택 프로파일을 12밴드 `DynamicsProcessing.PostEQ` gain으로 근사 적용.
- 플레이어 탭에 `처리 방식` 선택 UI 추가.
- 단위 테스트, debug/release 빌드, PD20 1회 설치 및 기본 재생 검증.

## 제외 범위

- 외부 앱 AudioEffect broadcast 세션 적용은 제외한다.
- `DynamicsProcessing` MBC/limiter/maximizer 고급 UI는 제외한다.
- Android EQ 세부 밴드 편집 UI는 제외한다.
- AutoEQ/OPRA 탭에 별도 메뉴를 추가하지 않는다.
- 시스템 전체 적용이나 session 0 global effect 사용은 제외한다.
