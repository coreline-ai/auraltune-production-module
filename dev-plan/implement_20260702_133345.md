# implement_20260702_133345.md

작성 일시: `2026-07-02 13:33:45 KST`

이 문서는 Android EQ 모드의 플레이어 청취 버튼을 `원음 / EQ 적용 / 프리앰프 적용` 3단계로 정리하고, 프로파일 헤드룸 적용을 사용자가 명확히 선택할 수 있게 만드는 개발 범위를 고정한다.

## 개발 목적

Android EQ 모드에서도 AuralTune 모드처럼 청취 상태가 버튼만 보고 명확히 이해되도록 만든다.  
기존 Android EQ 모드는 `EQ 적용` 버튼 하나가 `AUTOEQ`와 `USER`를 함께 선택된 것처럼 표시했고, 자동 헤드룸이 항상 적용되어 “EQ만 적용”과 “EQ+프리앰프”를 구분할 수 없었다. 이번 작업은 Android EQ에서 `USER` 모드를 `프리앰프 적용`으로 표시하고, 이때만 `DynamicsProcessing` input gain에 프로파일 `preampDB`를 적용한다.

## 개발 범위

| 구분 | 내용 |
|---|---|
| 대상 화면 | 플레이어 탭의 Android EQ 모드 컨트롤 |
| 버튼 구성 | `원음`, `EQ 적용`, `프리앰프 적용` |
| 상태 매핑 | `ORIGINAL = 무처리`, `AUTOEQ = EQ만`, `USER = EQ + 프로파일 preampDB 헤드룸` |
| 엔진 경로 | 기존 `DynamicsProcessing` PostEQ + input gain 경로 재사용 |
| 상태 표시 | Android EQ 상태 텍스트의 `헤드룸 x.x dB` 유지 |
| 테스트 | ViewModel/Controller 단위 테스트와 기존 instrumented smoke 회귀 확인 |

## 제외 범위

- AuralTune 자체 엔진의 `내 설정` 의미 변경 없음.
- 그래픽 EQ/파라메트릭 EQ 편집 UX 변경 없음.
- Android EQ에 리미터, MBC, 맥시마이저 추가 없음.
- 새로운 DataStore 키 또는 마이그레이션 추가 없음. 기존 `ListenMode.USER`를 Android EQ에서만 다른 라벨로 해석한다.
- 외부 앱 audio session 처리 없음.
