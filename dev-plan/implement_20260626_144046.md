이 문서는 현재 작업트리 기준 남은 작업의 범위를 고정하고, 출시 직전 정합화와 회귀 안전장치가 목적 밖으로 확장되지 않도록 하기 위한 작업 문서다.

## 개발 목적

AuralTune의 현재 릴리스 후보 상태를 기준으로 stale 문서, OPRA 상업 출시 문구, 로컬 release 검증, 오디오 라우트 선택 QA 증적, AutoEQ DB 회귀 테스트, 300+ delta 처리 정책을 정리하고 필요한 패치를 적용한다.

## 개발 범위

- 문서 SSOT 정합화: `remaining_20260622.md`, OPRA 계획서, 디자인 핸드오프의 오래된 TODO/상태를 2026-06-26 검증 기준으로 갱신한다.
- OPRA 상업 출시 문구: 앱 내 고지를 보완하는 EULA/OSS notice/스토어 설명 문구를 현재 채택 기준으로 문서화한다.
- 로컬 release 검증: release APK debug-marker scan, 16KB alignment, unit/release build를 Windows 로컬 스크립트로 재현 가능하게 유지한다.
- 실기기 오디오 라우트 QA: 선택 운영 증적으로 수동 검증 매트릭스 문서를 유지한다. 기능 완료 판단에는 포함하지 않는다.
- 라우트 정책 회귀 테스트: 물리 QA 전 단계에서 `DeviceKey`의 적격/부적격 출력 분류를 단위 테스트로 고정한다.
- AutoEQ DB 회귀 테스트: migration, ETag unchanged, malformed index 보존, stored filters/parser 동치성을 테스트로 보강한다.
- P3 테스트 세분화: `ActivityScenario.recreate()` 회전 테스트와 실제 malformed fixture 주입 테스트를 추가한다.
- 300+ delta 처리: GitHub compare cap 초과 시 full resync fallback 또는 명확한 UI/상태 정책을 코드와 테스트에 반영한다.

## 제외 범위

- T2-OS/T2-Custom/T3/HAL/Magisk 등 외부앱·시스템 전역 적용은 사용자 결정에 따라 제외한다.
- query-driven 온라인 fallback, bit-perfect/direct AudioTrack, 그래픽 EQ 기본 프리셋, 20밴드 Q 청취 튜닝은 이번 필수 패치 범위에서 제외하고 후속 선택 항목으로 유지한다.
- 법률 자문 자체는 수행하지 않는다. 상업 출시 문구는 현재 사용자 결정에 따라 채택된 고지 문구와 체크리스트로 제한한다.
