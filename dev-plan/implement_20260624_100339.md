## 개발 목적

기존 AutoEQ 기능을 1탭으로 유지하면서, OPRA 데이터를 2탭으로 분리 도입한다. OPRA 도입부는 기존 AutoEQ 데이터 경로에 직접 섞지 않고 `:opra-data` 단일 Gradle 모듈로 분리해 데이터 수집, 파싱, 캐시, 라이선스 메타데이터를 독립 관리한다.

| 목표 | 내용 | 판단 |
| --- | --- | --- |
| UI 분리 | AutoEQ 탭과 OPRA 탭을 분리해 검색, 목록, 선택 흐름을 독립 제공 | 중복 데이터와 출처 혼동을 줄이는 방향 |
| 모듈 분리 | `:opra-data` 신규 모듈에서 OPRA JSONL 수집, 파싱, 저장, 조회 담당 | 기존 `:autoeq-data` 변경 최소화 |
| 엔진 재사용 | OPRA 프로파일을 기존 DSP 엔진 입력 모델로 변환해 적용 | 네이티브 오디오 엔진 변경 최소화 |
| 상업 배포 대응 | OPRA/CC BY-SA 4.0 출처, 라이선스, 비공식 인증 고지를 앱에 노출 | 배포 리스크 감소 |
| 검증 가능성 | 파서, 필터 매핑, 캐시, 선택 복원, UI 탭 동작을 테스트로 분리 | 회귀 추적 가능 |

## 상업 배포 데이터 운영 정책

OPRA 공식 소비 문서는 상업 제품에 대해 `dist/` 폴더를 자체 인프라에 mirror/cache하도록 안내한다. 따라서 AuralTune의 release 앱은 OPRA GitHub를 실시간 데이터 서버처럼 사용하지 않고, AuralTune이 관리하는 배포 계층을 통해 검증된 snapshot만 소비한다.

시퀀싱: mirror 인프라 구축/운영은 본 계획의 제외 범위이므로, **첫 release는 bundled immutable snapshot 단독(라이브 업데이트 비활성)으로 출시 가능**하다. AuralTune mirror/cache가 준비된 뒤 update 경로를 활성화한다. 클라이언트는 처음부터 "mirror/snapshot만 사용" 정책으로 구현하되, mirror URL이 비어 있으면 bundled snapshot으로만 동작하도록 한다.

| 정책 항목 | release 기준 | 구현 반영 |
| --- | --- | --- |
| 원본 소스 | OPRA upstream `dist/database_v1.jsonl` 및 `dist/assets/...` | mirror job의 입력으로만 사용 |
| 앱 기본 URL | AuralTune 관리 mirror/cache URL | `BuildConfig` 또는 remote config의 release 기본값 |
| 금지 URL | `raw.githubusercontent.com`, `github.com/opra-project/OPRA` 직접 데이터 fetch | release 빌드 설정/테스트에서 차단 |
| Roon Labs mirror | 내부 검증, 비교, 비상업 참고용 | release 기본값으로 사용하지 않음 |
| bundled snapshot | 앱에 포함되는 마지막 검증 snapshot | 첫 실행, 오프라인, mirror 실패 시 fallback |
| snapshot manifest | `snapshot_version`, `opra_commit`, `generated_at`, `sha256`, `schema_version`, `license_url` | 업데이트 전 검증과 감사 추적 |
| 동기화 방식 | manifest 확인 → checksum 검증 → JSONL 파싱 → Room transaction 갱신 | 실패 시 기존 last-good cache 유지 |
| 버전 고정 | 앱 release마다 기준 snapshot을 기록 | 재현성, 문의 대응, 라이선스 고지 |
| 사용자 고지 | OPRA 출처, preset creator, CC BY-SA 4.0, no endorsement 표시 | OPRA 탭/상세/설정/OSS notice에 반영 |

### 권장 배포 흐름

```text
OPRA upstream repository
  -> scheduled AuralTune mirror job
  -> immutable snapshot storage + manifest/checksum
  -> schema/filter/license validation
  -> AuralTune CDN or bundled app asset
  -> :opra-data import/cache
  -> OPRA tab + attribution UI
```

### release 차단 조건

| 차단 조건 | 이유 | 검증 방식 |
| --- | --- | --- |
| release 기본 URL이 GitHub raw/API를 가리킴 | 상업 제품에서 GitHub를 데이터 서버처럼 사용하는 구조 | Gradle unit test 또는 build config assertion |
| release 기본 URL이 Roon Labs mirror를 가리킴 | 상업 제품 장기 운영 책임을 외부 mirror에 의존 | release variant 설정 테스트 |
| manifest checksum 불일치 | 데이터 변조/부분 다운로드 가능성 | import 전 checksum 검증 |
| OPRA commit/snapshot metadata 누락 | 출처 추적과 라이선스 고지 불충분 | sync state/entity 테스트 |
| attribution/license UI 누락 | CC BY-SA 조건 미충족 위험 | UI 체크리스트/스크린샷 검증 |

주의: URL 차단 검증은 **release 변종 기준**으로 해야 한다. `testDebugUnitTest`는 debug 변종이라 GitHub/raw URL이 허용값이므로, 차단은 **release `BuildConfig` 값에 대한 assertion 또는 release-variant 단위테스트**(예: `testReleaseUnitTest` 또는 변종 무관 상수 검증)로 확인한다.
