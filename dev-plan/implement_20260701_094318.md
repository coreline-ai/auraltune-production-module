# implement_20260701_094318.md

작성 일시: `2026-07-01 09:43:18 +09:00`

이 문서는 **플레이어 첫 섹션(Now-playing 패널)에 재생곡 앨범아트를 블러 처리한 고급스러운 배경**을 채우는 개발 범위를 고정한다. 앨범아트가 없는 곡은 번들 기본 이미지(`res/drawable/default_album_art.webp`, 사용자 제공)를 동일하게 블러 배경으로 쓴다. 가독성을 위해 어두운 스크림을 겹친다.

## 개발 목적
- 플레이어 홈의 **첫 섹션**(스펙트럼 + 곡 제목 + 보정 배지 + 시크바 + 트랜스포트가 든 `AuralTunePanel`, [AuralTuneApp.kt:363-433](../app/src/main/java/com/coreline/auraltune/ui/AuralTuneApp.kt#L363)) 배경을 **현재 재생곡 앨범아트의 블러 이미지**로 채워 프리미엄 감성 제공.
- 앨범아트가 없는 곡은 **기본 이미지(골드 마이크)** 로 대체.
- 어떤 아트에서도 텍스트·컨트롤 **가독성 유지**(다크 스크림).

## 개발 범위
- ExoPlayer 메타데이터에서 **임베디드 앨범아트 추출**(오프스레드 디코드 + 곡별 캐시) → `PlaybackUiState`에 노출.
- 첫 섹션 뒤에 **블러 배경 + 스크림** 레이어(신규 `BlurredArtBackground` 컴포저블).
- 아트 없음/디코드 실패 시 **기본 이미지** 폴백.
- 곡 전환 시 배경 **크로스페이드**.

## 제외 범위
- 미니플레이어·다른 화면(AutoEQ/OPRA)에는 적용하지 않음(플레이어 첫 섹션 전용).
- 앨범아트 편집/다운로드/온라인 조회 없음(임베디드 커버만).
- 새 이미지 라이브러리(Coil/Glide) 도입하지 않음 — `artworkData` ByteArray를 `BitmapFactory`로 직접 디코드.
- `default_album_art.webp` 파일 생성은 범위 밖(사용자가 배치). 없으면 그라디언트로 graceful fallback.

## 설계 결정 (기본값)
| 항목 | 결정 |
|---|---|
| 블러 방식 | **전 API 공통**: 입력 비트맵 소형 다운스케일(~48px) + 확대 bilinear 보간. **API 31+**: 추가로 `Modifier.blur(24.dp)`. |
| 스크림 | 세로 그라디언트(투명 상단 → `#000` 약 65% 하단) + 살짝 전체 다크 틴트. 텍스트 영역 가독성 우선. |
| 기본 이미지 | `res/drawable/default_album_art.webp`(사용자 제공). 없으면 브랜드 그라디언트. |
| 아트 소스 | `player.mediaMetadata.artworkData` (없으면 `MediaMetadataRetriever.getEmbeddedPicture(uri)` 폴백). |
| 캐시 | 현재 곡 키(mediaId/uri)별 1개 디코드본 캐시 — 위치 틱/재구성마다 재디코드 금지. |
| 전환 | 곡 변경 시 `Crossfade`. |
| 적용 대상 | 플레이어 첫 섹션(Now-playing 패널)만. |

## Phase 2. 블러 배경 컴포저블

### 목표
- 아트(또는 기본 이미지)를 블러+스크림으로 그리는 재사용 컴포저블을 만든다.

### 구현 태스크
- [ ] `BlurredArtBackground(artwork: ImageBitmap?, modifier, content: @Composable BoxScope.() -> Unit)`(신규):
  - `Box`(라운드 clip) → 배경 이미지(`ContentScale.Crop`, 섹션 채움) → 스크림 그라디언트 → `content()`.
- [ ] 블러: 입력 소형 다운스케일 + `FilterQuality.Low` 확대(전 API); **API 31+** 배경에 `Modifier.blur(24.dp)` 추가.

## 리스크 / 고려
- 가독성: 밝은 앨범아트에서 스크림 부족 시 텍스트 안 보임 → 스크림 농도 보수적으로. (튠 포인트)
- 성능: 디코드는 오프스레드, 블러 입력은 소형(~48px)이라 부하 미미. 캐시로 재디코드 방지.
- 메모리: 풀해상도 아트 보관 금지(다운스케일본만).
- API<31: `Modifier.blur`는 no-op이므로 다운스케일 트릭이 실제 블러를 담당(전 기기 동작).
- 리소스 부재: 기본 이미지 파일이 아직 없으면 그라디언트 폴백으로 개발/빌드 지속 가능.
