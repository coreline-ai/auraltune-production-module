# implement_20260702_131335.md — Android EQ 자동 헤드룸 설계/구현 계획

## 개발 목적

Android EQ 모드(`DynamicsProcessing`)에서도 AutoEQ/OPRA 프로파일의 `preampDB`를 자동 헤드룸으로 적용한다.  
현재 AuralTune 자체 엔진은 프리앰프를 별도 처리하지만, Android EQ 모드는 PostEQ 밴드만 적용하므로 큰 부스트가 포함된 프로파일에서 출력 헤드룸 확보가 불명확하다. 이번 작업은 사용자가 별도 프리앰프 버튼을 조작하지 않아도 Android EQ 모드가 프로파일 기준 감쇠값을 자동으로 반영하도록 만든다.

## 개발 범위

| 구분 | 내용 |
|---|---|
| 대상 모드 | 플레이어 내부 `Android EQ` 모드 |
| 적용 API | `android.media.audiofx.DynamicsProcessing.setInputGainAllChannelsTo()` |
| 입력 값 | 현재 active AutoEQ/OPRA 프로파일의 `preampDB` |
| 상태 전달 | `AutoEqViewModel → PlaybackProcessingState → AuralTuneMediaService → PlayerDynamicsEqController → OsEffectBackend` |
| UI 표시 | Android EQ 상태 텍스트에 자동 헤드룸 dB 표시 |
| 검증 | 단위 테스트, Android instrumented smoke test, 빌드 검증 |

## 제외 범위

| 제외 항목 | 사유 |
|---|---|
| 리미터/MBC/맥시마이저 추가 | 헤드룸 감쇠와 별도 DSP 정책이며 청취 검증 범위가 커짐 |
| Android EQ 프리앰프 토글 버튼 | 1차 목표는 단순한 자동 적용이며, 플레이어 메뉴 복잡도 증가 방지 |
| 외부 앱 audio session / session 0 처리 | 현재 제품 방향은 내부 플레이어 세션 전용 |
| Equalizer fallback의 입력 게인 보정 | Android `Equalizer`에는 동일한 input gain API가 없으므로 무시 |
| 실제 음량 측정 자동화 | 이번 범위는 API 전달/상태/연속 재생 smoke 검증 |

## 설계 원칙

1. Android EQ 모드는 자체 엔진과 상호 배타적으로 동작한다.
2. 자동 헤드룸은 프로파일의 `preampDB`를 그대로 사용하되, 양수 boost로 동작하지 않도록 0 dB 상한을 둔다.
3. 비정상 값(`NaN`, `Infinity`)은 0 dB로 보정한다.
4. 과도한 감쇠값은 native 프로파일 검증 범위에 맞춰 -30 dB 하한으로 제한한다.
5. UI는 버튼을 늘리지 않고 상태 텍스트에 “헤드룸 -x.x dB”만 표시한다.
