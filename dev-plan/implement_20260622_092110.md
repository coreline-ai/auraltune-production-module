## 🎯 목표 (Goal)

AutoEQ 헤드폰 보정 위에, **사용자가 직접 조정하는 그래픽 EQ**를 추가한다.
- **상단**: 현재 적용 EQ의 주파수 응답 곡선 그래프
- **하단**: 20밴드 슬라이더 (사용자가 조정 → 실시간 반영)
- 엔진의 **Manual chain(최대 20필터, peaking)** 을 채워서 동작

## 📐 확정 설계 (앞선 검증·논의 결과 반영)

| 항목 | 결정 | 근거 |
|---|---|---|
| 사용자 그래픽 EQ 엔진 | **Manual chain** (peaking, 최대 20) | 코드 확인: `kMaxManualFilters=20`, 현재 비어 있음(필터 0) |
| 밴드 수 | **20밴드** | 사용자 결정. Manual chain 풀 활용 |
| 밴드 주파수 | **로그 균등 20점 (20Hz~20kHz)**, 라벨은 반올림 표기 | `f_i = 20·1000^(i/19)` |
| 필터 타입/Q | peaking, **Q는 인접 밴드 −3dB 교차** 기준(≈2.9) 또는 고정 Q 검토 | Manual은 peaking only |
| AutoEQ 프로파일 | 그대로 유지(10필터, correction) | 실측 122개 100% 10필터 확정 |
| 둘의 관계 | **독립·합성** (Manual → AutoEQ 신호 순서) | 별개 cascade |
| 그래프 곡선 | **합성 곡선** (Manual + AutoEQ 응답 합) | "실제 귀에 들리는 결과" |
| 응답 계산(freqz) | **Kotlin에서 계산**(필터 파라미터 → RBJ → magnitude). 네이티브 무변경 | 필터 파라미터를 Kotlin이 모두 앎(밴드 게인 + 프로파일 필터). 그래프는 시각화라 Kotlin 계산으로 충분 |

### 신호 흐름
```
음악 → [Manual: 사용자 20밴드] → AutoEQ preamp → [AutoEQ: 프로파일 10필터] → ... → 출력
        ↑ 그래픽 EQ 슬라이더가 채움                  ↑ 헤드폰 보정(그대로)
그래프 = Manual 응답 + AutoEQ 응답  (합성, dB 합산)
```

## 📦 범위 (Scope)
- 20밴드 그래픽 EQ UI (슬라이더 + 게인 표시 + 리셋)
- 주파수 응답 그래프 (Canvas, 합성 곡선)
- 슬라이더 → `engine.updateManualEq` 실시간 연동 (debounce)
- Kotlin freqz 유틸 (RBJ magnitude 계산)

## 🚫 제외 (Exclusions)
- AutoEQ 프로파일 필터 직접 편집(프로파일은 읽기 전용 유지)
- 밴드 주파수/Q 사용자 커스터마이즈(고정 20밴드)
- 프리셋 저장/불러오기(후속)
- GraphicEQ.txt(101점)/FixedBandEQ 임포트(후속)
- 네이티브 엔진 변경(코어 무수정 — Kotlin freqz 사용)

## 📚 참조 (References)
- 실측 검증: AutoEq ParametricEQ = **10필터** (글로벌 브랜드 122개 100%), FixedBand=10, Graphic=101
- 엔진: `core/AuralTuneEQEngine.{h,cpp}` — Manual chain(20, peaking, `updateManualEq`), `peakingCoeffs` RBJ
- `AudioEngine.kt` — `updateManualEq(frequencies, gainsDB, qFactors)`, `setManualEqEnabled`, `MAX_MANUAL_FILTERS=20`
- `AGENTS.md` — 코어 무수정/단일 writer 원칙

## 📐 공통 규칙 (Common Rules)
1. **코어 무수정**: 네이티브 DSP 변경 없음. 그래프 응답은 Kotlin freqz로 계산.
2. **단일 writer 주의**: AutoEQ writer는 `DeviceAutoEqManager`. Manual writer는 **그래픽 EQ 전용 경로**가 단독 소유(서로 다른 chain이라 충돌 없음).
3. **RT-safe 연동**: 슬라이더 → debounce(≈30~50ms) → `updateManualEq`(atomic publish). 매 프레임 호출 금지.
4. **빌드 검증**: 항상 gradlew **exit code**로 확인(`tee` 파이프 exit 오인 금지).
5. **peaking only** 제약 인지: 모든 밴드는 peaking(양끝도 shelf 아님).
