# implement_20260629_145803.md

작성 일시: `2026-06-29 14:58:03 +09:00`

이 문서는 AutoEQ/OPRA 프리앰프 토글 및 프로파일 교체 시 발생할 수 있는 즉시 게인 점프를 엔진 레벨에서 완화하는 작업 범위를 고정한다.

## 개발 목적

AutoEQ ON/OFF의 0.5초 wet/dry 크로스페이드는 유지하되, 프리앰프 게인 변경 경로에도 짧은 per-frame 글라이드를 추가해 프리앰프 버튼 토글과 프로파일 교체 시 음량이 즉시 튀는 현상을 제거한다.

## 개발 범위

- 네이티브 `AuralTuneEQEngine`의 AutoEQ preamp 적용 경로에 오디오 스레드 소유 글라이드 상태를 추가한다.
- 프리앰프 타깃은 `preampOn ? autoEqPreampLinear : 1.0f`로 계산한다.
- 첫 오디오 콜백은 타깃으로 스냅해 기존 골든 테스트 결정성을 유지한다.
- 이후 타깃 변경은 약 30ms 동안 선형 보간한다.
- AutoEQ wet/dry 크로스페이드와 독립적으로 동작하게 한다.
- 프리앰프 토글 회귀 테스트를 추가한다.

## 제외 범위

- Kotlin API 변경 없음.
- UI/문구 변경 없음.
- AutoEQ ON/OFF 0.5초 크로스페이드 길이 변경 없음.
- loudness, limiter, manual EQ 동작 변경 없음.
- 프리앰프를 별도 사용자 설정값으로 재설계하지 않음.
