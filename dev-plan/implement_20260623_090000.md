## 🎯 목표
"카탈로그(목록)만 seed + 본체 on-demand" → **"카탈로그 + 모든 파싱 프로파일을 prebuilt DB로 번들"** 로 확장.
- 검색→선택→**적용까지 전부 즉시·오프라인**(첫 선택도 네트워크 0).
- 업스트림 변경 시 **전체 재처리 없이 추가/변경분만**(delta) 로컬화.

## 📐 확정 설계
| 항목 | 결정 | 근거 |
|---|---|---|
| seed 범위 | 전체 catalog(6,028) + profiles(6,027) + filters(60,270) | 사용자 결정(전체) |
| 적재 방식 | **Room `createFromAsset`** (prebuilt `autoeq_seed.db`) | 첫 실행 네트워크/파싱 0, 즉시 |
| 스키마 | v2 그대로(identityHash `5674e7e3…`) — **delta 커밋은 `sync_state` 행(key=`autoeq_commit`)에 저장**해 스키마 변경 회피 | createFromAsset 호환 유지 |
| 생성기 | `tools/gen_autoeq_seed.py` — 앱 파서(Index/ParametricEq/id/url)를 **정확 복제**, 병렬 다운로드, schemas/2.json에서 스키마+identityHash 추출 | id/필터가 앱과 동일해야 DB-first 적중 |
| 증분 갱신 | **GitHub compare API** (`/compare/{local}...{HEAD}`) → 변경 `ParametricEQ.txt`만 fetch/upsert, removed 삭제, INDEX 재파싱으로 catalog 정합 | 전체 재처리 회피 |

### 실측 크기 (정정)
- INDEX 원본 8,854줄 → **dedup 후 고유 6,028개** (8,854는 줄 수였음).
- 생성 DB: **11.65MB**(VACUUM 후) / APK 내 **deflate 3.45MB(70%)** / 전체 debug APK 26.2MB.
- 프로파일 1개 원본 ~480B(필터 10), 파싱본만이면 작지만 SQLite/인덱스/문자열 오버헤드로 DB는 11.65MB.

## 🚫 트레이드오프(정직)
- 제 초기 "+1~2MB" 추정은 틀림 → 실제 **APK +~3.45MB / 설치 후 DB ~11.65MB**.
- staleness: 스냅샷. 단 AutoEq 깃이 ~11개월 정적(HEAD `7ae0f56…`, 2025-07-20)이라 위험 낮음 + delta로 보강.
- 대안(미채택): 런타임 JSON seed → APK ~0.5MB로 작지만 첫 실행 ~2-4s seed. (원하면 전환 가능)
