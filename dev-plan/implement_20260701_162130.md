# implement_20260701_162130.md

작성 일시: `2026-07-01 16:21:30 +09:00`

두 가지를 함께 구현한다: **(A) 재생곡 앨범아트에서 강조색을 추출(Palette)해 슬라이더·트랜스포트 버튼 색에 반영**(곡마다 커버에 맞춰 변함), **(B) 재생목록(큐) 행 왼쪽에 트랙별 앨범아트 썸네일 표시**. 기존 아트 추출 파이프라인(`implement_20260701_155744.md`)을 재사용/확장한다.

## 개발 목적
- (A) now-playing 슬라이더(thumb/active)·재생 버튼·셔플/반복 "on" 틴트를 현재 곡 커버의 강조색으로. 곡 전환 시 부드럽게 애니메이션.
- (B) `QueueRow` 왼쪽 아이콘 자리를 **트랙별 앨범아트 썸네일**로. 현재 곡 행은 아트 위에 재생/일시정지 오버레이.

## 개발 범위
- `androidx.palette:palette-ktx` 추가.
- `rememberArtworkAccent(artwork, fallback)`: 현재 곡 `state.artwork`(512px)에서 Palette 강조색 추출(오프스레드) + 가독성 가드 + `animateColorAsState`.
- 강조색 적용: 시크 슬라이더, 재생 `FilledIconButton` 컨테이너(+대비 content color), 셔플/반복 활성 틴트.
- `AlbumArtCache`(앱 싱글턴): URI별 소형(128px) 썸네일 디코드 + LRU 캐시(오프스레드).
- `rememberTrackArtwork(cache, uri)`: 큐 행에서 비동기 로드.
- `QueueRow`: 왼쪽에 40dp 아트 썸네일(무아트=기본커버), 현재 곡은 재생/일시정지 오버레이.

## 제외 범위
- EQ 모드 칩(원음/EQ적용/커스텀/프리앰프)·상단바·탭 등 플레이어 트랜스포트 밖 색은 변경 안 함(강조색은 슬라이더+트랜스포트 한정).

- 큐 행 아트는 소형 썸네일만(대형/리오더/저장 없음).

## Phase 4. 강조색 견고화 (원인 분석 + 수정)

### 문제
- 08/02/03/05 등 커버가 있는데도 강조색이 테마색(틸)으로 폴백되는 곡이 다수.

### 원인 (초판 로직의 폴백 3종)
- ② **휘도 가드 과다**: `luminance() >= 0.20` 하드 컷 → 진한 파랑/빨강/보라 등 흔한 채도색이 상대휘도 0.20 미만이라 대거 탈락(틸·밝은색만 통과). 가장 큰 원인.
- ③ **후보 협소**: `vibrant ?: lightVibrant ?: dominant`만 → 저채도/파스텔에서 vibrant null → 부정확.
- ① **무아트**: `artwork==null`이면 Palette 전에 즉시 테마색(기본 커버에서 안 뽑음).

### 구현 태스크
- [x] ② 하드 휘도 컷 제거 → `liftForDarkSurface()`로 **HSL 명도만 0.55로 상향**(hue/채도 유지). 어두운 커버색도 살림.
- [x] ③ 후보 확대: `vibrant ?: lightVibrant ?: darkVibrant ?: muted ?: lightMuted ?: darkMuted ?: dominant`.
- [x] ① `artwork==null`이면 **기본 커버(골드마이크) 비트맵에서 추출**(1회 디코드 캐시) → 무아트 곡도 색 반영.
- [x] 폴백은 "디코드 불가/스와치 전무"인 진짜 실패에만.
