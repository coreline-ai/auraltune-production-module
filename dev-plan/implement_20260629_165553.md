# implement_20260629_165553.md

작성 일시: `2026-06-29 16:55:53 +09:00`

이 문서는 OPRA 프로파일을 사용하는 상태에서도 바로 적용 가능한 파라메트릭 EQ 시작점 프리셋 기능의 개발 범위를 고정한다. 목표는 OPRA 값을 수정하거나 OPRA 데이터로 새 파생 보정을 만드는 것이 아니라, 기존 OPRA/AutoEQ 보정 위에 사용자가 선택하고 조절할 수 있는 앱 자체 파라메트릭 Manual EQ 시작점을 제공하는 것이다.

## 개발 목적

- OPRA 프로파일 선택 후 `내 설정`에서 사용자가 빠르게 출발점을 고를 수 있는 파라메트릭 EQ 프리셋을 제공한다.
- 프리셋은 "검증된 전문가 보정"이 아니라 "시작점"으로 표기하고, 선택 후 사용자가 주파수/게인/Q/타입을 조절할 수 있게 한다.
- 프리셋 선택 시 기존 파라메트릭 밴드 목록을 교체하고, 그래프 점/곡선/엔진 Manual chain이 즉시 일치하게 한다.
- AutoEQ/OPRA 탭별 선택, active provider, OPRA attribution/license 고지와 충돌하지 않게 한다.

## 개발 범위

- 파라메트릭 EQ 전용 기본 시작점 프리셋 13개를 앱 코드에 정적 카탈로그로 추가한다.
- 사용자 저장 파라메트릭 프리셋을 DataStore에 저장/불러오기/삭제할 수 있게 한다.
- 기본 프리셋은 삭제/덮어쓰기 불가, 수정본은 사용자 프리셋으로 저장한다.
- 프리셋 선택 후 사용자가 밴드를 수정하면 `수정됨` 또는 `커스텀` 상태로 표시한다.
- OPRA가 active provider일 때도 `내 설정 = OPRA 프로파일 + 현재 파라메트릭 프리셋/수정값`으로 동작하게 한다.
- 그래프는 기존 composite 응답 계산을 사용해 OPRA/AutoEQ 프로파일 곡선과 파라메트릭 Manual EQ 곡선을 함께 반영한다.
- 기존 native Manual chain과 `AudioEngine.updateManualEq()` 경로를 그대로 사용한다.

## 제외 범위

- OPRA 데이터 값 수정, OPRA 프로파일 재생성, OPRA-derived preset 생성은 하지 않는다.
- 새 DSP 필터 타입(Low Pass, Band Pass, Notch 등)은 추가하지 않는다.
- 그래픽 EQ와 파라메트릭 EQ를 동시에 Manual chain에 적용하지 않는다.
- 파라메트릭 밴드 수 상한은 현재 UX 기준 `ParametricBand.MAX_BANDS = 8`을 유지한다.
- `ParametricEQ.txt` import/export는 이번 범위에서 제외한다.
- "전문가 인증", "검증된 프리셋", "게임 최적화" 같은 확정/인증성 문구를 사용하지 않는다.

| UI 문구 | `OPRA + FPS 시작점`, `OPRA + 커스텀`처럼 소스와 사용자 EQ를 분리해 이해되게 표시한다. |
| 검증 문구 | 기본 프리셋은 `시작점`, `추천 시작점`으로만 표기한다. |

## 데이터 모델 초안

| 모델 | 필드 | 설명 |
|---|---|---|
| `ParametricEqPreset` | `id`, `name`, `category`, `source`, `bands`, `createdAtMillis`, `updatedAtMillis` | 기본/사용자 프리셋 공용 표시 모델 |
| `ParametricPresetSource` | `BUILT_IN`, `USER` | 기본 프리셋과 사용자 저장 프리셋 구분 |
| `ParametricPresetBand` | `type`, `freqHz`, `gainDb`, `q` | `ParametricBand`에서 id를 제외한 저장 단위 |
| `ParametricPresetSelection` | `presetId`, `source`, `dirty` | 현재 선택/수정 상태 |

구현 시 `ParametricPresetBand`를 별도로 두거나 기존 `ParametricBand`를 재사용할 수 있다. 단, 기본 프리셋은 고정 id가 아니라 적용 시 새 band id를 생성해 편집 충돌을 피한다.
