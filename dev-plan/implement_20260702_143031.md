# implement_20260702_143031.md

작성 일시: `2026-07-02 14:30:31 +09:00`

기본 데모 곡(`천상연_이창섭.m4a`)을 **APK 에셋으로 번들**하고, 저장된 큐가 없을 때(첫 실행/큐 비움) **실행 시 자동 로딩**한다. 앱이 항상 재생할 곡을 갖게 해 첫인상·데모를 개선한다.

## 개발 목적
- `app/src/main/assets/cheonsangyeon.m4a`(4.4MB)를 번들.
- MusicPlayerController가 복원할 스냅샷이 없으면 이 에셋을 기본 트랙으로 큐에 로딩(자동재생 없이 일시정지 노출 — 기존 restore 정책과 동일).
- 에셋 곡도 커버 아트·가수/앨범이 정상 표시되게(art-centric UX 일관).

## 개발 범위
- 에셋 배치 + `noCompress "m4a"`(MediaMetadataRetriever `openFd` 가능하도록).
- `ArtworkDecoder`/`AlbumArtCache`가 `asset://` URI를 처리(AssetFileDescriptor로 열기).
- `MusicPlayerController.restore()`: 스냅샷 없음/빈 큐 → `asset:///cheonsangyeon.m4a`(제목 "천상연_이창섭.m4a") 기본 로딩.

## 제외 범위
- 여러 데모 곡/플레이리스트 없음(1곡).
- 에셋을 항상 강제 선두 삽입하지 않음 — **스냅샷이 있으면(사용자 큐) 그대로 복원**, 없을 때만 기본 로딩.
- 재생 파이프라인/EQ/처리모드 변경 없음. asset은 ExoPlayer AssetDataSource로 재생(기존 경로).
- SAF 권한(persistRead)은 asset에 불필요 — restore 경로는 원래 persistRead 호출 안 함.
